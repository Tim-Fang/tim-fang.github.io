开源协议简介

Apache License 2.0 (Apache-2.0)

你可以:

  • 修改代码需要在被修改的文件中作必要的说明
  • 使用或改编源代码必须附带一份原有的 Apache 协议
  • 使用或改编可以不使用 Apache License 开源

BSD License 3-clause (BSD-3-Clause)

你可以:

  • 使用或改编源代码必须附带一份原有的 BSD 协议
  • 不可以用开源代码的作者/机构名称和原来的产品名称做市场推广

GNU Affero General Public License version 3 (AGPL-3.0)

你可以:

  • 使用或改编都必须使用 AGPL-3.0 开源

GNU General Public License version 3 (GPL-3.0)

你可以:

  • 使用或改编都必须使用 GPL-3.0 开源

备注: 鉴于 Scratch 的特殊性, GPL 和 AGPL 的差别无法体现, 建议使用 AGPL 开源

MIT License (MIT)

你可以:

  • 使用或改编并在需要时修改开源协议
  • 必须包含版权声明和许可声明

Creative Commons License (CC)

CC0 协议

你可以:

  • 没有限制

CC 4.0 International 协议

下文括号中的表示条件, 不需要在声明中出现.

CC 协议应该是由 BY, NC, ND, SA 四种条件的任意组合, 其中 ND 和 SA 不可同时出现, 缺乏 BY 的不再使用.

协议简称实例: CC BY-NC-SA 4.0

你可以:

  • 使用或改编本作品
  • 必须给出适当的署名并提供许可协议链接 (如果有 BY)
  • 不得作为商业用途 (如果有 NC)
  • 不得发布使用或改编自本作品的作品 (如果有 ND)
  • 必须使用相同的协议发布作品 (如果有 SA)
  • 使用法律术语或技术措施限制其他人做许可协议允许的事情
